# Election Analysis: Nepal’s 2079 Parliamentary Elections

This project provides a data-driven analysis of Nepal’s 2079 parliamentary elections, specifically focusing on generating actionable insights for alternative parties and developing strategic campaigning models.

---

## Project Overview

The repository contains tools and data to analyze election results across various constituencies to help political strategists and alternative parties understand voting patterns and improve their electoral performance.

---

## Repository Structure

The project is built entirely using **Jupyter Notebooks (100%)** and includes the following key files and data:

### 1. Data Processing
- **Changing_data_to_csv.ipynb**: This notebook is responsible for the initial data cleaning and transformation, converting raw electoral data into usable CSV formats.

### 2. Election Data

The repository includes several datasets containing constituency-specific results:

- **all_constituencies.csv**: A consolidated dataset of electoral results.  
- **constituency_1.csv** to **constituency_10.csv**: Detailed individual data files for specific constituencies (1 through 10).

### 3. Core Analysis
- **data_analysis.ipynb**: The primary notebook where the election data is analyzed to extract insights regarding party performance and strategic trends.

---

## Key Features

- **Alternative Party Insights**: Targeted analysis to help newer or non-traditional parties understand their standing.  
- **Strategic Campaigning**: Data-backed suggestions for campaign optimization based on the 2079 election results.  
- **Constituency Breakdown**: Granular data access for at least 10 different electoral constituencies.

---

## How to Use

1. **Data Preprocessing**: Run `Changing_data_to_csv.ipynb` if you need to re-process raw data into CSV format.  
2. **Analysis**: Open and run `data_analysis.ipynb` to view the visualizations and insights derived from the constituency data.
